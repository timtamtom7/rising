# Muse — R9: Post-Launch, Monetization Refinement, Roadmap

## Goal
Analyze launch performance, refine the business model, and plan the next development cycle.

---

## Scope

### Launch Metrics
- Downloads, revenue from App Store Connect
- Retention: day 1/7/30 active users
- Crash rate vs beta
- IAP conversion rate (free → Pro)
- Average session length
- Top features by usage

### Business Model Refinement
- Review pricing: is Pro subscription too high/low?
- Consider family sharing for Pro tier
- Consider lifetime license option
- Student discount (.edu verification)
- Referral program: give free month for each referral

### Feature Roadmap (Next Cycle)
Based on reviews and analytics:
1. **Sleep timer**: auto-pause after X minutes
2. **Lyrics**: synced lyrics display (via LRCLIB or similar)
3. **Smart speaker integration**: Sonos, Chromecast Audio
4. **Audiobooks**: separate mode with chapters and bookmarks
5. **Crossfade UI redesign**: visual preview of crossfade zone

### Maintenance
- macOS update compatibility within 1 week of each beta
- Xcode update within 1 week
- Security patches within 48 hours of CVE
- Keep dependencies updated (SQLite.swift, etc.)

### Community
- GitHub discussions for feature requests
- Changelog blog per release
- Social media presence (X account for updates)

---

## Out of Scope (R10)
- End-of-life planning
