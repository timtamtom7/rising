# fume — Round 12: Social Features, Shared Libraries, Collaboration

## Goal
Add social and collaborative features to Fume — enabling shared knowledge libraries, collaborative annotation, and a community of knowledge collectors.

---

## Scope

### Shared Libraries (Pro Tier)
- **Create a shared library**: Invite up to 10 people to a shared knowledge library
- **Collaborative annotation**: Anyone in the library can add notes, highlights, comments to shared sources
- **Permission levels**: Owner (can delete library), Editor (can add/annotate), Viewer (read-only)
- **Activity feed**: See what others in your library have been saving and annotating
- **Library themes**: "Philosophy library", "Startup research", "Design references"

### Community Discovery
- **Public libraries**: Option to make a library public (read-only for non-members)
- **Library directory**: Browse public libraries by topic — find "Philosophy for Engineers", "Startup Mental Models"
- **Featured libraries**: Curated by Fume team — best public libraries in each category
- **Follow libraries**: Subscribe to a public library and get notified of new sources

### Collaborative Features
- **Real-time co-annotation**: See collaborators' highlights and comments appear in real-time
- **Discussion threads**: Start a discussion about a specific source or paragraph — threaded replies
- **Quote-of-the-day from libraries**: Get a daily notification with an interesting quote from a library you follow
- **Source recommendations**: "Users with similar libraries also saved..." — collaborative filtering

### Privacy Controls
- Per-library privacy settings (private, invite-only, public)
- Individual source sharing: share a single source via link (no account needed to view)
- Anonymize before sharing: blur all personal notes before sharing
